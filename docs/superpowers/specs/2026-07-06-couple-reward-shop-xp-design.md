# 情侣奖励商城与 XP 打卡系统设计

## 状态

- 日期：2026-07-06
- 状态：已确认
- 范围：单机本地优先版本，不引入账号、云同步或后台管理系统
- 目标用户：女朋友作为日常使用者；你作为奖励配置与兑现确认者

## 背景

当前 App 已具备习惯创建、AI 计划预览、每日打卡、连续天数、月历记录、提醒、阶段调整建议、主题和数据导出能力。下一阶段希望增强“打卡后的即时动力”和“坚持后的真实回报”，让用户每次完成习惯都能获得 XP，并用 XP 在奖励商城兑换虚拟奖励或现实奖励。

这个功能不是面向大众用户的商业商城，也不是多人后台系统。第一版应保持亲密、轻量、本地优先：她负责坚持和兑换，你负责配置奖励和确认兑现。

## 目标

1. 用户每次完成有效打卡后获得 XP，并看到清晰反馈。
2. XP 有余额、累计获得量和流水记录，避免只靠一个数字难以追踪。
3. 商城同时支持 App 内虚拟奖励和现实奖励。
4. 普通使用者只能浏览和兑换奖励，不能新增、编辑或下架奖励。
5. 你可以通过隐藏管理入口维护奖励，并确认现实奖励已经兑现。
6. 兑换现实奖励后进入“待兑现”，由你确认后变为“已兑现”。
7. 第一版不做扣分、惩罚、排行榜、账号系统和远程同步。

## 非目标

- 不做多用户账号、登录、管理员后台或云端权限系统。
- 不做支付、真实商品库存、物流、优惠券码或第三方商城。
- 不做复杂游戏经济，例如等级树、装备、抽奖、盲盒、连续惩罚。
- 不允许普通使用者添加自定义现实奖励。
- 不把 XP 绑定到单个习惯；第一版使用全局 XP 钱包。

## 核心体验

### 普通使用者路径

1. 用户在今日页完成一个习惯。
2. 系统写入打卡记录，并计算本次获得的 XP。
3. 页面展示轻量反馈，例如“本次 +10 XP，当前余额 230 XP”。
4. 用户进入商城，查看虚拟奖励与现实奖励。
5. 当 XP 足够时，用户点击兑换。
6. 虚拟奖励兑换后立即生效；现实奖励兑换后进入“待兑现”。
7. 用户可以在兑换记录中查看“待兑现、已兑现、已取消”的记录。

### 管理者路径

1. 你通过隐藏入口进入管理模式。
2. 输入管理 PIN 后进入奖励管理页。
3. 你可以新增、编辑、下架奖励，设置名称、描述、价格、类型和展示状态。
4. 你可以查看待兑现记录，并确认“已兑现”或取消兑换。
5. 确认兑现只改变兑换状态，不重复扣 XP。

## XP 规则

第一版 XP 规则保持简单且偏正向激励。

| 场景 | XP |
| --- | ---: |
| 普通打卡完成 | +10 |
| 连续 3 个应执行日完成 | +20 |
| 连续 7 个应执行日完成 | +50 |
| AI 计划阶段完成 | +100 |
| 中断后重新回来完成 | +15 |

规则说明：

- 不扣 XP，不因断签惩罚。
- 同一个习惯同一天只允许获得一次普通打卡 XP。
- 连续奖励只在达成当日发放一次，避免重复打开页面反复领取。
- “回来奖励”用于鼓励中断后的首次恢复，不和惩罚绑定。
- XP 计算应依赖打卡历史和交易流水，保证幂等。

## 商城奖励类型

### 虚拟奖励

虚拟奖励适合较低价格和即时反馈，例如：

- 主题色
- 打卡庆祝动效
- 称号
- 徽章样式
- 今日页卡片皮肤

虚拟奖励兑换后状态直接为“已兑现”，同时写入拥有记录。第一版可以先只支持“购买并记录已拥有”，具体主题或动效是否立即应用可按实现成本拆分。

### 现实奖励

现实奖励由你配置，例如：

- 奶茶一杯
- 一起看她选的电影
- 你洗碗一次
- 周末约会基金
- 一个大惊喜

现实奖励兑换后状态为“待兑现”。你确认后改为“已兑现”。如果取消兑换，第一版建议退回 XP，并写一条退款流水。

## 页面设计

### 今日页

新增内容：

- 打卡完成后展示本次 XP 反馈。
- 已完成习惯可以展示今日已获得 XP。
- 今日页顶部或完成反馈中显示当前 XP 余额。

交互要求：

- 打卡和 XP 发放必须是同一个事务语义：不能打卡成功但 XP 丢失。
- 如果重复点击已完成习惯，不应重复获得 XP。

### 我的页

新增内容：

- XP 余额卡片：当前余额、累计获得。
- 商城入口。
- 兑换记录入口。
- 隐藏管理入口。

隐藏入口建议：

- 在“我的”页标题或 XP 卡片上连续点击 5 次后出现 PIN 输入。
- 管理 PIN 本地保存。首次进入管理模式时设置 PIN。
- 这是本地使用隔离，不声明为高强度安全机制。

### 商城页

展示：

- 当前 XP 余额。
- 奖励列表。
- 奖励类型标签：虚拟、现实。
- XP 价格。
- 可兑换、XP 不足、已下架、已拥有等状态。

操作：

- XP 足够且奖励可用时允许兑换。
- 现实奖励兑换后提示“已提交兑换，等待兑现”。
- 虚拟奖励兑换后提示“已解锁”。

### 兑换记录页

分组：

- 待兑现
- 已兑现
- 已取消

每条记录展示：

- 奖励名称
- 消耗 XP
- 兑换时间
- 状态
- 兑现时间或取消时间

### 管理页

能力：

- 新增奖励。
- 编辑奖励名称、描述、类型、价格、状态。
- 下架奖励。
- 查看待兑现记录。
- 确认兑现。
- 取消兑换并退回 XP。

普通用户路径中不展示新增奖励入口。

## 数据模型

### xp_wallet

记录全局 XP 钱包。

- id
- balance
- lifetime_earned
- lifetime_spent
- updated_at

第一版只需要一条钱包记录。

### xp_transactions

记录所有 XP 变动。

- id
- amount
- type: earn、spend、refund、adjust
- reason: checkin、streak_3、streak_7、plan_complete、return_bonus、reward_redeem、redemption_cancel
- habit_id
- check_in_id
- reward_id
- redemption_id
- date_key
- created_at

约束：

- 对于打卡 XP，使用 habit_id + date_key + reason 做幂等保护。
- 所有余额变化必须先写流水，再更新钱包。

### rewards

记录奖励配置。

- id
- title
- description
- type: virtual、real_world
- price_xp
- status: active、archived
- virtual_kind: theme、celebration、title、badge、card_skin、none
- inventory_limit
- created_at
- updated_at

第一版 inventory_limit 可为空，不做复杂库存。

### reward_redemptions

记录兑换。

- id
- reward_id
- price_xp
- status: pending_fulfillment、fulfilled、cancelled
- created_at
- fulfilled_at
- cancelled_at
- note

规则：

- 虚拟奖励兑换后直接 fulfilled。
- 现实奖励兑换后 pending_fulfillment。
- cancelled 时退回 XP。

### admin_settings

记录本地管理设置。

- key
- value

需要支持：

- admin_pin_hash
- admin_enabled

PIN 不明文保存。可以使用轻量哈希；第一版本地隔离即可，不承诺抵御设备级攻击。

## 数据流

### 打卡获得 XP

1. 今日页调用现有打卡逻辑。
2. 打卡成功后调用 XP 发放服务。
3. XP 服务读取当前习惯、当天日期、历史打卡和已有流水。
4. XP 服务计算本次应发放的交易列表。
5. 写入 `xp_transactions`。
6. 更新 `xp_wallet`。
7. 返回本次获得 XP 明细给 UI。
8. UI 展示反馈。

### 兑换奖励

1. 商城页读取钱包余额和 active 奖励。
2. 用户点击兑换。
3. 兑换服务校验余额是否足够、奖励是否可兑换。
4. 写入 spend 流水并扣减钱包。
5. 创建兑换记录。
6. 虚拟奖励直接 fulfilled；现实奖励 pending_fulfillment。
7. UI 展示结果。

### 取消现实奖励

1. 管理页选择取消兑换。
2. 兑换服务校验记录处于 pending_fulfillment。
3. 状态改为 cancelled。
4. 写入 refund 流水并退回 XP。
5. 更新钱包。

## 错误处理

- XP 不足：阻止兑换，显示还差多少 XP。
- 奖励已下架：阻止兑换，提示奖励不可兑换。
- 重复打卡：不重复发放 XP。
- 重复兑换点击：按钮进入 loading；服务层再次校验余额。
- 取消已兑现奖励：不允许取消，除非后续设计人工调整入口。
- 管理 PIN 错误：提示错误，不展示管理页面。

## 测试策略

### 单元测试

- XP 普通打卡只发放一次。
- 连续 3 天、7 天奖励只在达成日发放一次。
- 中断后回归奖励正确触发。
- XP 余额不足不能兑换。
- 现实奖励兑换后为待兑现。
- 虚拟奖励兑换后为已兑现。
- 取消待兑现奖励会退回 XP。
- 已兑现奖励不能取消。

### 集成验证

- 从今日页打卡后，XP 余额增加且反馈展示。
- 商城可用余额正确影响兑换按钮状态。
- 兑换现实奖励后，在兑换记录和管理页都可见。
- 管理页确认兑现后，普通用户记录变为已兑现。

## 验收标准

1. 用户完成一次有效打卡后获得 XP，且刷新后余额不丢失。
2. 同一习惯同一天重复打开或重复点击不会重复获得 XP。
3. 商城显示虚拟奖励和现实奖励。
4. 普通用户不能在普通界面新增、编辑、删除奖励。
5. 管理 PIN 通过后可以维护奖励。
6. XP 足够时可以兑换奖励；不足时不能兑换。
7. 现实奖励兑换后进入待兑现。
8. 管理者确认后兑换记录变为已兑现。
9. 管理者取消待兑现兑换后 XP 退回。
10. 不需要账号、网络或后台服务即可完成第一版闭环。
