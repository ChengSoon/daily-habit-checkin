# 体素冒险地图（3D 沙盘）设计文档

日期：2026-07-11
状态：已与用户逐节确认
取代：`2026-07-10-3d-star-island-map-design.md`（WebP 场景拼接方案，全部作废）

## 背景与目标

现有冒险地图由多张 1080×1920 的 WebP 场景图纵向拼接、SVG 叠加路线构成。用户要求重做为**一个真正的 3D 地图**：不是图片拼接，而是实时渲染的立体世界，场景中的元素（水、云、树、角色、建筑）持续动态。

经浏览器实时原型对比，用户确认的方向：

| 决策点 | 结论 |
|---|---|
| 镜头交互 | 沙盘式：单指平移 + 双指缩放，固定俯视角（约 42°），不可旋转 |
| 美术风格 | 体素方块风（Crossy Road 式），延续现有粉彩童话配色 |
| 世界生长 | 群岛链：每 4 关一座浮岛，云雾中浮现新岛，无限延伸 |
| 页面形态 | 冒险 Tab 全屏地图，信息模块悬浮叠加 |
| 渲染引擎 | @react-three/fiber v9 (native) + expo-gl + three.js，程序化体素生成 |
| 平台 | iOS + Android；Web 显示简化占位 |
| 旧拼接版 | 工作区未提交的拼接地图改动直接丢弃 |

**成功标准**：中端安卓机 ≥ 45fps；两台设备渲染出完全一致的世界；关卡无限增长时地图规则无需改动；GL 失败时功能可降级不丢失。

## 一、总体架构与页面结构

冒险 Tab 即一整屏体素世界。原信息模块改为悬浮层：

- **顶部悬浮条**：章节名、累计行动力胶囊
- **底部抽屉（可上滑）**：下一站奖励预览、旅程收藏（AdventureCollection）、站点管理入口（OwnerGate 保护）
- **角落按钮**："回到当前节点"

### 模块划分

| 模块 | 职责 | 测试 |
|---|---|---|
| `src/adventure/voxelWorldLayout.ts` | 纯函数：stations + progress → 岛屿/桥/章节门/预告岛/路线的 3D 坐标与状态 | vitest |
| `src/adventure/voxelIslandRecipes.ts` | 纯函数：岛序号 + 章节主题 → 立方体摆放清单（含上限断言） | vitest |
| `src/adventure/VoxelWorldCanvas.tsx` | r3f Canvas 渲染层：消费上述纯函数输出，组织 InstancedMesh | 真机 verify |
| `src/adventure/useMapCamera.ts` | 手势 → 相机平移/缩放，边界钳制逻辑抽纯函数 | vitest（钳制） |
| `src/adventure/useUnlockCeremony.ts` | 解锁仪式时序编排（纯函数）+ 执行 hook | vitest（时序） |
| 悬浮层组件 | 复用现有行动力/奖励/收藏/管理组件 | 现有测试 |

### 数据流

不变：campaign stations 顺序是唯一数据源，`floor(stationIndex / 4)` 定岛、`floor(stationIndex / 12)` 定章节，不修改服务端与数据库。`pendingUnlockStationIds` 仍是解锁仪式的触发入口。

## 二、体素世界构建

### 布局

- 世界铺在 XZ 平面，旅程沿 -Z（"北方"）延伸，岛屿左右交错成蜿蜒群岛链
- **岛**：圆形体素平台（草地层 + 泥土层 + 底部岩石，直径约 14 格），岛内小径串起 4 个关卡节点
- **桥**：相邻岛之间的体素吊桥
- **章节门**：每 3 座岛（12 关）之间的体素拱门；未通过时门扇关闭、门后浓雾
- **预告岛**：进度前方第一座未解锁岛，云雾半掩、轮廓朦胧
- **云海**：世界边缘留白，容纳无限生长

### 程序化 + 确定性

每座岛由"配方"生成：以岛序号为随机种子，从章节主题模板（春樱 → 夏湖 → 秋林 → 雪山 → 星空，循环）确定树木、花丛、岩石摆放和一座地标建筑（城堡/灯塔/风车/温室，呼应现有 landmark 语义）。同一种子输出恒定 → 跨设备一致 + 可单测断言。

### 路线与角色

- 石板小径 TubeGeometry 贯穿全程
- 节点：已通过 = 粉色发光圆台；当前 = 金色 + 脉冲光环；未来 = 灰色石台
- 情侣化身：两个体素小人，各用其主题色，依偎在当前节点旁轻轻浮动

### 资源策略

零模型文件、零纹理：全部立方体 + 纯色材质，程序化生成，彻底规避 RN 端 GLTF/纹理加载管线（技术调研确认的最大痛点）。全场景按材质合并 InstancedMesh。

## 三、镜头手势与动画

### 镜头

- 透视相机，固定俯视角约 42°，禁止旋转
- 单指拖动平移（带惯性阻尼），钳制在"已解锁群岛 + 预告岛"包围盒
- 双指捏合缩放：近至看清小人动作，远至一章 3 座岛尽收眼底
- 进入页面镜头落在当前节点；"回到当前"按钮平滑飞回
- 点击节点：镜头轻推对准 + 节点信息卡（关卡名、门槛、奖励预览）
- 手势状态经 ref 直达渲染循环，不触发 React 重渲染

### idle 动画（全场景常动）

云朵漂移、水面方块起伏、树木摇曳、旗帜摆动、当前节点光环脉冲、化身浮动并偶尔对视。灯光随章节主题变化（晨光/日暮/月夜循环）；App 深色模式切夜景。`useReducedMotion` 开启时动画减半或静止。

### 解锁仪式

1. 镜头从当前位置飞向目标节点
2. 跨岛时：云雾散开，新岛方块自下而上逐层升起（体素版"岛屿苏醒"）
3. 双人化身沿小径蹦跳前往新节点，跨过吊桥
4. 节点点亮 + 彩带粒子 + 奖励摘要卡（复用现有奖励语义）
5. 任意手势可跳过，直接到终态；reduced motion 下直接淡入终态

时序编排（阶段/时长/插值目标）为纯函数，渲染层只执行。

## 四、性能、降级、错误处理与测试

### 性能预算

- 三角形 ≤ 100k，draw call ≤ 50（十几个 InstancedMesh + 少量独立 mesh）
- 配方生成时断言每岛方块数上限
- 窗口化：镜头附近 ±2 岛全细节；远岛仅平台 + 地标剪影；更远不挂载
- DPR ≤ 2；离开 Tab 暂停渲染循环
- 帧率自适应：连续低帧 → 先关阴影 → 再减装饰实例

### 错误处理

- Canvas 外包 ErrorBoundary：GL 初始化失败/崩溃 → 文字版进度卡（章节 + 关卡列表 + 当前位置）
- 世界零远程资源，断网无影响
- Web 平台：不渲染 3D，直接显示简化占位（文字版进度卡）

### 测试

- `voxelWorldLayout`：0/1/4/5/11/12/13/24/48/120 关的岛数、门数、预告岛、坐标单调性
- `voxelIslandRecipes`：同种子确定性、方块数上限、无重叠
- 解锁仪式时序、镜头钳制：纯函数断言
- GL 渲染层真机 verify，不进单测

### 迁移与依赖

- 新增依赖：`three`（锁定小版本）、`@react-three/fiber@^9`、`expo-gl@~57`、dev 依赖 `@types/three`（均已核实活跃维护：r3f 9.6.1 / three 0.185.1 / expo-gl 57.0.1，2026-07 更新）
- 丢弃未提交拼接版改动前，先甄别：属于拼接地图的（场景 manifest、SceneSegment、FogLayer、WebP 资源等）丢弃；属于站点管理等无关功能的保留
- 移除 `assets/adventure-map/*.webp` 与场景 manifest
