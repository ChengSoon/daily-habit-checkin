# 卡卡 App 内受控动作设计

## 状态

- 日期：2026-07-21
- 状态：已确认，实施中
- 范围：把卡卡从纯聊天升级为可执行白名单 App 动作的受控 Agent

## 目标与边界

第一版支持完成打卡、新建习惯、修改习惯名称/提醒/频率、暂停或恢复习惯。所有写操作先生成结构化提案，由发起成员确认后执行。

不支持删除、积分消费、奖励兑换、管理员操作、任意 HTTP/数据库/Shell 工具。模型不直连数据库，不能绕过确认。

## 流程

```text
用户消息
→ 服务端动作规划器
→ Zod 校验的白名单动作
→ 持久化 pending 提案
→ 客户端确认卡
→ 服务端重新校验空间、发起人、状态、过期时间和目标记录
→ 事务执行
→ 同步业务资源与共同对话
```

## 安全与一致性

- 待确认动作绑定 `space_id` 和 `requested_by`，15 分钟过期。
- 确认使用行锁和状态机，重试不重复执行。
- 打卡与 XP 在同一事务中完成，之后发布 `check_ins`、`wallet`、`adventure` 和 `companion` 同步失效。
- 模型返回非法动作时降级为普通聊天，不执行任何写入。
- 卡卡的文字不得在执行成功前声称操作已完成。
