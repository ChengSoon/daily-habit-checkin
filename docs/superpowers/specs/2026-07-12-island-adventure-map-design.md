# 岛屿闯关地图设计

## 状态

- 日期：2026-07-12
- 状态：Phase 1+2 已落地（Stitch 默认主题岛 + 可上传岛图 + 自动延伸）
- 范围：世界地图视觉与交互范式从「路径节点」升级为「解锁岛屿」
- 前置：线性章节解锁、程序化布局、管理端 `nodeImageKey` 上传已存在
- 参考：用户提供的群岛地图（竖向海洋 + 多座立体岛 + 序号解锁感）
- 不改：`lifetimeEarned` 线性解锁、领奖幂等、双人 space 共享进度

## 一句话目标

把世界地图做成 **可滚动的群岛海图**：每一章 = 一座可点击的「立体」岛屿；达标解锁一座；锁定岛呈剪影/雾；管理端可自上传每座岛的美术图；加章后岛屿自动排布延伸，无需手调坐标。

## 你要的体验（对齐参考图）

| 参考图特征 | 产品映射 |
| --- | --- |
| 竖向大地图、海洋背景 | 可平铺/可延伸的海天背景 |
| 每座立体岛屿 | 一章一岛（透明 PNG 岛图 + 阴影） |
| 按进度「出现」 | 已解锁彩色岛 / 锁定灰剪影或云雾 |
| 可点 | 点岛 → 章节详情 / 领奖 |
| 自己的风格 | 管理端上传 `islandImage`（复用/演进自 node 图） |

**重要澄清：不需要真 3D 引擎。**  
手游里的「立体岛」≈ **2.5D 插画**（带透视的 PNG/WebP + 阴影 + 轻微浮动动画）。React Native 用 `Image` + `Svg` 即可，成本可控。

## 目标

1. 视觉上是「闯关岛屿地图」，不是圆点路径图。
2. 每解锁一章，对应岛从锁定态变为可交互彩色岛（可加出现动画，二期）。
3. 管理端可为每章上传自定义岛图；无图时用主题默认岛。
4. 章节数增加时，岛屿自动排布、画布自动变高。
5. 解锁/领奖业务规则零回归。

## 非目标（本期）

- 不做 Unity / Three.js 真 3D 岛。
- 不做岛上内部关卡小地图（一章仍是一个里程碑）。
- 不做自由拖拽摆岛 CMS（默认算法布局；可选 override 二期）。
- 不做多线分支航线（仍线性 sortOrder）。
- 不强制用户必须上传岛图（有默认皮肤）。

## 信息架构

```
Space 共享进度
  └─ Chapter[n]  （业务：门槛 XP、叙事、徽章）
       └─ Island visual  （表现：岛图、布局位、锁定态）
```

业务层保持 `adventure_chapters`；表现层把「节点」重命名为「岛屿」。

## 视觉分层（从底到顶）

```
1. OceanSkyLayer     海天背景（平铺竖图或渐变+波纹）
2. DecorLayer        轻云、远岛剪影（可选，纯装饰）
3. RouteLayer        岛屿间虚线航线（细，不抢戏）
4. IslandLayer       各岛精灵（主交互）
5. HUD / Caption     岛名、序号、状态胶囊
```

## 岛屿状态机（表现）

| viewStatus | 表现 |
| --- | --- |
| `locked` | 灰阶/剪影 + 低透明；可点（进详情看门槛）或不可点进故事全文（建议可点看门槛）；序号锁 |
| `claimable` | 彩色岛 + 呼吸光圈/金边 + 「可领取」角标 |
| `claimed` | 彩色岛 + 已领小旗/徽章角标；航线连到下一岛 |

当前推进中的下一座锁定岛可略亮于更后面的锁岛（「雾散下一座」），增强闯关感。

## 布局：程序化群岛（自动延伸）

仍用纯函数布局，**不是**死底图钉点。

```
// i = 0..N-1，第 1 章靠下（近海启程），往后往上飘
canvasH = PAD_TOP + PAD_BOTTOM + max(N-1,0) * ISLAND_GAP
x = 0.5 + side * AMP + slight noise   // 左右错落，像群岛
cy = H - PAD_BOTTOM - i * ISLAND_GAP
islandScale = baseScale * (1 - i * 0.02) // 可选：远处略小，增强纵深
```

参数建议：

- `ISLAND_GAP` ≈ 140–160（比路径节点更大，岛要占地）
- `AMP` ≈ 0.20–0.24
- 岛显示宽约 110–140dp；阴影 offset 向下

加第 N+1 章：只多一个 layout 槽位 + 自动加高，**不改旧岛坐标算法输入**（坐标由 index 决定，旧岛位置稳定——注意：若 gap 固定，旧岛像素位置在高度变化时会整体上移，这是可接受的滚动地图行为；若需「已探索区域坐标绝对稳定」，二期可改为锚在底部累加，仅顶部延伸）。

**坐标稳定性策略（本期采用）**：

- 以 **底部为锚**：`cy = H - PAD_BOTTOM - i * GAP`，`H` 随 N 增大时，**第 1 章始终贴底区**，新岛往上长。
- 用户滚到底仍是启程岛；新岛在上方出现。符合「航线向上探索」。

## 岛图资产规范

### 上传图（用户/管理端）

| 项 | 要求 |
| --- | --- |
| 格式 | PNG（**必须透明背景**）或 WebP |
| 构图 | 单岛居中，略带 2.5D 透视；底部可留海崖/沙滩 |
| 尺寸 | 建议 512×512 或 768×768；最大边 ≤ 1024 |
| 禁止 | 整张带大海背景的全景（那是背景层，不是岛） |
| 安全区 | 主体在中心 80% 内，避免被圆形裁切（我们默认不裁圆，矩形透明） |

### 默认岛包（无上传时）

按 `mapThemeKey` 内置 6 套默认岛（本地 assets 或远程）：

`lighthouse / forest / market / camp / bridge / summit`

超出主题键：循环默认包或中性「无名岛」。

### 背景

- 一张竖向可平铺的海洋背景（或海→天渐变 + 简单波纹 SVG）。
- 与岛图分离：**背景可拼，岛可换**。

1. 文案与预览：大图预览岛，提示透明底。

## 与当前路径地图的关系

| | 路径节点图（现状） | 岛屿群岛图（目标） |
| --- | --- | --- |
| 主元素 | 圆点 + 线 | 岛精灵 + 弱航线 |
| 底图 | 大地图插画或渐变 | 海洋背景 + 独立岛资产 |

## 分阶段交付

### Phase 1 — 可玩的群岛感（建议先做）

1. `IslandMarker`：大尺寸岛图 + 阴影 + 三态。
2. 布局 gap 加大，底部锚点自动延伸。
3. 海洋背景层（可用现有 procedural 竖图裁/压暗作海，或纯海渐变）。
4. 管理端文案改为岛屿形象；继续写 `nodeImageKey`。
5. 默认：无上传时用 emoji/主题色底+标题，或 1 张通用默认岛 PNG。

### Phase 2 — 更像参考图

1. 6 套默认主题岛 assets。
2. 锁定剪影更精致；claimable 浮动动画（Reanimated 已有依赖）。
3. 解锁瞬间：云散/缩放弹入（本地动画，不改服务端）。

### Phase 3 — 运营向

1. `map_x/map_y/island_scale` override。
2. Region 群岛分区标题。
3. 用户侧「共创上传」（权限模型另定）。

## 成功标准

1. 打开世界地图，第一眼是「海 + 多座岛」，不是圆点折线。
2. 锁定 / 可领 / 已领三态岛可一眼区分。
3. 管理端上传透明岛图后，该章显示自定义岛。
4. 新增第 7 章后上方自动多一座岛，无需改代码坐标。
5. 章节详情与领奖流程与现在一致。

## 风险

| 风险 | 缓解 |
| --- | --- |
| 用户上传含背景的全景图 | 管理端说明 + 预览衬底；后期可加「去底」但不做自动抠图 |
| 岛太大重叠 | gap 与 amp 单测；caption 限宽 |
| 包体变大 | 默认岛 WebP；用户图走 R2 不进包体 |
| 「不够 3D」预期 | 文档与 UI 说明为 2.5D 插画；Phase2 加阴影与浮动 |

## 开放问题（请拍板）

1. **锁定岛是否可点？** 推荐：可点，进详情只展示门槛与模糊故事。
2. **无自定义岛图时：** A 通用默认岛 / B 按 mapThemeKey 多套默认 / C 仅 emoji 占位？推荐 B（可先 A 再补 B）。
3. **航线要不要？** 推荐保留很淡的虚线，增强「下一座在哪」。

## 结论

这种「解锁一座岛、岛立体可点、可上传样式」的正确设计是：

> **2.5D 岛屿精灵 + 程序化群岛布局 + 海洋背景层 + 复用 nodeImageKey 作为岛图上传**  
> 而不是真 3D，也不是一张死大地图手钉坐标。

业务仍是线性 XP 里程碑；变的是地图表现层，与情侣打卡目标兼容。

## 2.5D 立体岛实现要点（已落地）

手游「立体岛」= 假 3D，真 3D 不必要。本项目采用：

1. **分层体积**：SVG 椭圆叠 投影 → 深崖 → 浅崖 → 顶面 → 高光  
2. **接触阴影**：岛底扁椭圆，像立在海面上  
3. **可领浮动**：Reanimated 上下 6dp 循环  
4. **自定义上传**：`nodeImageKey` 透明 PNG，底下仍加阴影与锁定遮罩  
5. **主题色岛**：无图时按 `mapThemeKey` 程序化上色  

升级到「更立体」的路径（未做）：

- 换手绘/AI **带透视的透明岛 PNG**（收益最大）  
- 增加第二层装饰（小屋、树）作为子 Image  
- 不做 Three.js，除非要旋转观察

## 交互修订（单岛焦点）

- 一屏只展示当前一座岛（不再同时铺满群岛路径）。
- 上下滑动（分页 FlatList）切换上一岛 / 下一岛。
- 默认定位到：可领取岛 → 否则最高已解锁 → 否则第 1 岛。
- 岛体持续轻浮动（锁定弱 / 已解锁中 / 可领强）；去掉代码双重阴影。
- 加章后列表自动变长，滑动即可到达新岛（自动延伸）。

## 自定义与动态图（已落地）

- `nodeImageKey`：每岛自定义形象，支持 GIF 动画（上传不经 JPEG 压缩）。
- `backgroundImageKey`：每岛自定义背景，支持 GIF；缺省用默认海图。
- `mapThemeKey`：无自定义岛图时的默认主题岛皮肤。
- 上传 MIME：jpeg/png/webp/**gif**；adventure 上限 8MB。
