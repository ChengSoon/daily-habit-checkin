# 卡卡桌宠设计

## 目标

把现有由基础 SVG 图形拼成的“卡卡”替换为有真实材质、完整动作和可拖动交互的应用内桌宠，同时保留已有打卡反馈与 AI 对话能力。

## 视觉

- 角色：软乎乎的 3D 毛绒海豹“卡卡”。
- 识别特征：两只清晰的小圆毛绒耳、奶白肚皮、短绒灰白身体、珊瑚色脸颊、青绿色针织项圈、无文字的金色圆牌。
- 风格：柔和工作室级 3D 玩偶渲染，材质真实但轮廓简洁，确保在约 90px 宽度下仍清楚可辨。
- 禁止：文字、Logo、地面阴影、漂浮特效、光晕、动作线、跨帧元素和背景残留。

## 动画资产

使用 `openai-image-api` 的 `gpt-image-2` 生成基准形象及动作条，使用 `hatch-pet` 的确定性脚本完成抠图、逐帧提取、图集组合和校验。

最终图集遵循固定契约：

- `1536x1872` WebP
- `8x9` 网格
- 单元格 `192x208`
- 九种状态：`idle`、`running-right`、`running-left`、`waving`、`jumping`、`failed`、`waiting`、`running`、`review`
- 未使用单元格完全透明

卡卡的项圈和圆牌保持左右对称，因此 `running-left` 可以在确认右行动画后逐帧镜像生成。

## 应用集成

- `PetSprite` 用透明 WebP 图集渲染真实逐帧动画，不再绘制 SVG。
- 现有情绪映射为图集状态：待机→`idle`，开心→`jumping`，思考→`running`，等待输入→`waiting`，错误→`failed`，招呼→`waving`。
- 用户拖动卡卡时，根据拖动方向即时切换 `running-left` / `running-right`。
- 点击卡卡仍打开现有 AI 对话面板；打卡、里程碑、AI 思考和错误仍通过 `PetContext` 驱动状态。
- 尊重系统“减少动态效果”设置：固定展示每个状态的首帧。

## 边界

本次不修改 AI 接口、数据库、同步协议、依赖、根配置或后端。宠物位置仅保存在当前会话，避免引入持久化迁移。

## 验收

1. 图集尺寸、透明度、使用帧和空帧通过 `hatch-pet` 校验。
2. 联系表和九组 GIF 经视觉检查，无角色漂移、裁切、背景残留或动作语义错误。
3. 应用中卡卡可逐帧播放、拖动、点击对话，并正确响应打卡与 AI 状态。
4. 定向测试、全量测试、TypeScript、ESLint 与 Web 端烟雾测试通过。
